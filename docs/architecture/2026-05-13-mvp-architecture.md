# MVP architecture notes

Date: 2026-05-13

## Scope

The implementation originally followed `docs/APPLICATION_DESIGN.md` as a Go CLI and later moved through Node.js + TypeScript. As of 2026-05-18, the current implementation is Rust with Cargo-built native executables for release distribution. The CLI remains thin:

- `src/main.rs` owns the binary entrypoint.
- `src/app.rs`, `src/cli.rs`, `src/config.rs`, `src/wizard.rs`, `src/lang.rs`, `src/prompt.rs`, `src/agent.rs`, and `src/output.rs` split CLI orchestration, parsing, config, setup, language aliases, prompt construction, Agent CLI execution, and output normalization.
- `tests/cli.rs` covers the public CLI shape and fake Agent CLI execution.
- `testdata/fake-codex/codex` and `testdata/fake-claude/claude` provide deterministic Agent CLI fixtures.
- `xtask/src/main.rs` packages Cargo release binaries into release archives, checksums, and the Homebrew formula.

## Adapter boundary

The CLI layer resolves the selected tool by ID and then uses the Adapter interface. Codex/Claude-specific command flags stay inside their adapters.

Codex uses:

- `codex exec`
- short prompt argument passing with stdin fallback for large prompts
- read-only sandbox
- approval disabled
- color disabled
- ignored user config and project rules
- ephemeral session
- Spark model with low reasoning
- JSON event streaming with early return on the final agent message

Claude uses:

- `claude -p`
- `--bare`
- JSON output
- JSON schema argument
- no session persistence
- one turn
- empty tools list

## Config format

The MVP keeps a small TOML reader/writer instead of adding a dependency. It supports the schema required by the design:

- `version`
- `default_tool`
- `local_lang`
- `timeout_ms`
- `[tools.codex] enabled`
- `[tools.claude] enabled`

The parser intentionally ignores unknown sections and keys to leave room for future config expansion.

## Testing strategy

Unit tests cover parsing, language aliases, prompt snapshots, config persistence, output normalization, errors, and adapter command construction.

Agent integration tests use `testdata/fake-codex/codex` and `testdata/fake-claude/claude` so CI never calls real Agent CLIs.

## Distribution

GitHub Release archives are generated by the tag release workflow with the design-specified asset names:

- `t-darwin-arm64.tar.gz`
- `t-darwin-amd64.tar.gz`
- `t-linux-amd64.tar.gz`
- `t-linux-arm64.tar.gz`
- `t-windows-amd64.zip`

The Homebrew formula installs the compiled `t` executable from the release archive and has no Node.js dependency. npm distribution is not part of the Rust-only implementation in this repository; adding it later should use a binary package strategy that does not reintroduce a JavaScript CLI implementation.
