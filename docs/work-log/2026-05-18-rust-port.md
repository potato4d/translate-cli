# Rust port

Date: 2026-05-18

## Objective

Convert translate-cli from the project-root Node.js / TypeScript implementation to a Rust implementation while preserving the MVP CLI behavior from `docs/APPLICATION_DESIGN.md`.

## Acceptance checklist

- `t --version` prints `translate-cli 0.1.1`.
- `t <text>`, `t <lang> <text>`, `--tool`, `--setup`, `--no-wizard`, and stdin input keep the existing CLI shape.
- Config remains the same TOML schema: `version`, `default_tool`, `local_lang`, `timeout_ms`, `[tools.codex]`, and `[tools.claude]`.
- Codex still runs through `codex exec` with read-only sandbox, approval disabled, ignored user config/rules, ephemeral execution, low reasoning, Spark model, prompt arg for short inputs, stdin fallback for large inputs, and JSON event streaming.
- Claude still runs through `claude --bare -p` with JSON output, JSON schema, no session persistence, one turn, and no tools.
- Output normalization accepts `translated_text`, `structured_output.translated_text`, nested `result`, embedded JSON, and raw text fallback.
- Tests use fake Agent CLIs under `testdata/` and do not call real Codex or Claude.
- Release artifacts remain named `t-darwin-amd64.tar.gz`, `t-darwin-arm64.tar.gz`, `t-linux-amd64.tar.gz`, `t-linux-arm64.tar.gz`, and `t-windows-amd64.zip`.
- Homebrew Formula installs a compiled `t` binary and does not depend on Node.js.
- Repository source code no longer contains TypeScript, JavaScript build scripts, `package.json`, `package-lock.json`, or `tsconfig.json`.

## Change

- Added a Cargo package with binary name `t`.
- Split the Rust implementation by responsibility:
  - `src/app.rs` orchestration
  - `src/cli.rs` argument parsing and usage
  - `src/config.rs` config read/write
  - `src/wizard.rs` first-run setup
  - `src/lang.rs` language aliases and locale detection
  - `src/prompt.rs` prompt construction
  - `src/agent.rs` Codex/Claude adapters and subprocess execution
  - `src/output.rs` result normalization
- Added Rust unit tests and integration tests in `tests/cli.rs`.
- Added `xtask` for release archive packaging, checksums, and Homebrew formula generation.
- Removed the TypeScript implementation and Node/Bun build files.
- Updated CI and release workflows to use Cargo.
- Updated README and architecture notes to describe the Rust implementation.

## Verification

- `cargo test --workspace` passed: 10 unit tests, 4 integration tests, and the xtask crate test target.
